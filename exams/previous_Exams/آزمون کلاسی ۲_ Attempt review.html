متن سؤال

کدام خط‌مشی (policy) توزیع بار بر اساس تقسیم دامنه فایل (file space) مابین خدمتگزارها (servers) کار می‌کند؟ مشکل ان چیست؟ کدام خط‌مشی این مشکل را حل می‌کند؟ (۳ دقیقه)

a.

خط‌مشی Cache Affinity، ممکن است بعضی از خدمتگزارها بواسطه میزبانی فایل‌های پرطرفدار (more popular) بار (load) بسیار بالایی داشته باشند، Cache Manager

b.

خط‌مشی CAP (Client-Aware Policy)، حجم خوبی از فایل‌های در حافظه اصلی (ram) خدمتگزارها خواهند بود و سریعتر می‌توان درخواست برای انها را پاسخ داد، Cache Manager

c.

خط‌مشی Cache Affinity، ممکن است بعضی از خدمتگزارها بواسطه میزبانی فایل‌های پرطرفدار (more popular) بار (load) بسیار بالایی داشته باشند، LARD (Locality aware request distribution)

d.

خط‌مشی SITEA (Size Interval Task Assignment with Equal Load)، حجم خوبی از فایل‌های در حافظه اصلی (ram) خدمتگزارها خواهند بود و سریعتر می‌توان درخواست برای انها را پاسخ داد، Cache Manager

متن سؤال

فرض کنید که برای توزیع بار از الگوریتم Weighted Round Robin استفاده می‌شود که وزن‌دهی را بر اساس میزان استفاده از CPU در خدمتگزارها (servers or nodes) انجام می‌دهد. کدام گزینه، وزن‌های (weights) صحیح‌تری برای چهار خدمتگزار شکل زیر است (به ترتیب از سمت راست به چپ برای خدمتگزار ۱ تا ۴)؟ (۴ دقیقه)

-خدمتگزارها تعداد یکسانی هسته CPU دارند و مدل CPU نیز برای آنها یکسان است.

-میزان استفاده شده از CPU برای هر خدمتگزار نمایش داده شده است.
